# Contributing to Meshmonitor

Thank you for your interest in contributing to the Meshmonitor Home Assistant add-on!

## How to Contribute

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes
4. Test your changes thoroughly
5. Submit a pull request

## Development Setup

To set up a development environment:

1. Clone this repository
2. Make your changes to the add-on
3. Test locally using the Home Assistant Supervisor

## Coding Standards

- Follow Python PEP 8 style guidelines
- Write clear, descriptive commit messages
- Add comments for complex logic
- Update documentation as needed

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub with:
- A clear description of the problem
- Steps to reproduce (for bugs)
- Expected vs. actual behavior
- Your Home Assistant version and architecture

## Pull Request Process

1. Ensure your code follows the project's coding standards
2. Update the README.md and CHANGELOG.md if needed
3. The PR will be reviewed by maintainers
4. Address any feedback from reviewers
5. Once approved, your PR will be merged

Thank you for contributing!
